Contributing to Iron Coder

In this tutorial, we will cover how to create a custom board for Iron Coder.
If this is something that interests you, great! Iron Coder, as an open source
project, welcomes collaboration and we are eager to see it grow from
community effort.

With that being said, lets get started. The first step is to clone the
repository, and navigate to the folder titled iron-coder-boards, as shown here:

We notice nested folders of format Manufacturer/Board/source files. When
creating your own board, follow this layout, as that is where the application will
search. As of writing this tutorial, Iron Coder supports Adafruit and Sparkfun
boards, but this is by no means a restriction! More boards are planned, and you
are welcome to develop for Manufacturers not listed.

The best way to develop your own board is to open one of the given boards and emulate
their format but with a custom board's contents. One of the notable artifacts necessary
for an Iron Coder board is the SVG file, which is a layered image format used for
determining where the pins are in the connection menu. There are many tools online, but
the provided boards used InkScape for this purpose.

There are some prerequisites for the formatting of the SVG file. These guidelines should
be sufficient:

  1. 1. Ensure that the image size is not too large.
  2. 2. The units should be in mm.
  3. 3. All transforms have been removed and the units of all elements are absolute.
  4. 4. All paths that should be displayed in Iron Coder have element IDs equal to those in the board pinouts.

For more specifics as to how SVG files are read in Iron Coder, refer to the source code at "src/board/svg_reader.rs". Once the above has been ensured, place the corresponding SVG file in the associated template folder of the corresponding board. The egui library will then be able to parse the contents of the file for proper display in Iron Coder.

This concludes the tutorial. We wish you the best of luck, and invite you to open a pull request!
